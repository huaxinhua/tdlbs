商管2.0

实现技术：

本框架使用vue + vuex + vue route + element + sass 实现

规约要求：

样式）

所有的组件重复样式要写在 components/TDConponents/style 目录下

所有页面的重复样式要写在 src/styles/tdstyle.scss 目录下

代码)

简单页面直接使用驼峰命令格式，在其大功能目录下(src/view/category/page.vue)

对于复杂页面，在其分类页面下新建以其功能命名的文件夹(src/view/category/xxx)，在文件夹内新建components文件夹和index.vue，

components用于存放复杂页面分离出来的部分功能

api目录和view目录遵循以业务模块区分，新建模块同名文件夹，在其文件夹内新建页面和api接口，详情见代码api/dictionaries示例

td-page

该组件为根组件，控制页面整体样式，默认只展示查询按钮，如需导出，添加needExport属性即可

td-page里存放五个具名插槽分别为

search 主要搜索条件

more 次要搜索条件

action 操作按钮

content 列表内容

drawer 侧边栏

td-page @query 搜索按钮点击事件

td-page @export 导出按钮点击事件

td-page - v-loading 统一控制页面加载状态，页面无须关心其值，从vuex中获取，页面只需在computer监听即可 (已更新组件自动控制，业务无须关心，如个别业务需要手动添加loading，自行处理)

td-page showDrawer 控制是否显示侧边栏

td-page - @closeDrawer 关闭侧边栏的回调，当设置autoClose属性时，点击侧边栏之外会直接关闭，不在询问

td-page - drawerTitle 侧边栏标题，不传则不显示标题

td-page autoClose 点击侧边栏之外的遮罩层，侧边栏会直接关闭，不会出现询问

td-page drawerSize 设置侧边栏宽度，默认30% 400px，仅支持传入百分比 像素值

侧边栏showDrawer不设置时，页面不加载侧边栏

次要搜索条件插槽和操作插槽可不写，页面自动适应

搜索条件和操作栏可不写，页面自动适应

search-view

该组件为搜索条件的根组件

搜索分为主要搜索条件和次要搜索条件，搜索条件高度默认70px，在components/TDConponents/style/style.scss中TDSearchRowHeight 定义

搜索条件一行能显示的数目会自动适应

主要搜索条件和次要搜索条件公用此组件，但主要搜索条件需要添加main属性

次要搜素条件默认不展开，展开高度会根据 具名插槽more 来进行自适应

search-item

该组件为搜索条件的根组件，必须包裹在search-view组件内，可包含多个

在内可添加input，select等组件，需要展示label时，添加title属性即可，默认宽度200px，宽度不够时可添加width属性设置宽度即可，后面的操作框会相应缩小

不需要显示label时，会自适应占满父组件宽度

td-table 表格组件

table-list 列表数据

header 规约表格内容，prop为需要显示字段，lable为表头内容，formatter(row)回调函数可对返回值进行特殊处理，建议在获取数据时进行处理，保证data中格式简约，render(h,data)回调函数可获取行内容

同时可进行其他属性设置

select 是否开启多选框

pagination 是否开启分页，开启分页之后需要实现@pageChange方法来进行翻页事件

在el-table-column中同样支持sortable,fixed,tooltop，此属性需要在设置table-list之前，保证数据列表存在该字段

@selection-change 开启多选之后，选中会返回选中的条目内容数据

商户/门店/品牌/签约人/承租单元 选择组件

组件目录：src/components/TDComponents/interactiveBox/index.vue

组件使用步骤

//引入组件
import storeBox from "@/components/TDComponents/interactiveBox/index.vue";

//组册组件
components: {
   storeBox
},

//template 使用组件
  <store-box :dialogState="isDialogState" @diloagClose="closeStore" :dialogType="dialogType"/>

closeStore(value) {
   if (!value) {
       this.isDialogState = value;
   } else {
       this.isDialogState = false;
       this.dialogType == "store" ? (this.selectStore = value) : (this.selectMerchant = value);
   }
},

属性介绍：

dialogState 控制组件显示隐藏

dialogType 控制组件类型：目前有门店/商户/品牌/签约人/承租单元（store/merchant/brand/signer/unit）五种类型

closeStroe 弹窗关闭事件：可通过事件返回值判断选中门店/商户/品牌/签约人/承租单元，并进行赋值
